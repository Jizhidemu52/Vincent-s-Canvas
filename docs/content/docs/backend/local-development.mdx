# 本地开发

业务功能依赖正式后端。只启动 `web/` 可以调整登录页和界面，但不能绕过 Session 使用生成、额度或素材接口。

## 推荐：Docker Compose

```bash
cp .env.example .env
# 替换 .env 中所有 replace-with 值
docker compose up -d --build
docker compose ps
curl http://localhost:3000/api/health
```

访问：

```text
设计师入口：http://localhost:3000/login
管理员入口：http://localhost:3000/admin/login
```

首次管理员登录后必须修改初始密码。开发环境需要模拟生成时可暂时设置 `TASK_MOCK_MODE=true`，它仍会走 Redis 队列、额度冻结、Worker、对象存储、历史和结算，只是不调用外部模型。

## 分开启动前端

```bash
cd web
bun install
bun run dev
```

Vite 会把 `/api` 代理到 `http://localhost:3100`，因此还需要启动 API、Worker、PostgreSQL 和 Redis。模型 Base URL 与 API Key 由超级管理员在后台配置，设计师端没有密钥弹窗。

## 质量检查

```bash
cd server
bun run build
bun test

cd ../web
bun run build
bun test
```

推送到 `main` 后，GitHub Actions 还会运行真实 PostgreSQL/Redis 集成、Compose 恢复演练、40 人并发和跨账号隔离测试。
