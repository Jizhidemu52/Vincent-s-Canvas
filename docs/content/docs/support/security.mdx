# 漏洞提交

请不要在公开页面、演示环境、共享截图或普通任务备注中直接发布漏洞细节、可利用代码、私密 API Key、真实用户数据或内部访问地址。

## 提交流程

发现安全问题后，请通过公司内部安全流程提交，并附带可复现信息。管理员收到后应先确认影响范围，再安排修复、回归验证和审计记录。

## 建议提供的信息

- 受影响的版本、提交、分支或部署环境。
- 清晰的复现步骤。
- 漏洞影响、触发条件和可能的数据范围。
- 已脱敏的日志、截图或 PoC。
- 是否涉及浏览器本地存储、API Key、WebDAV 同步、模型渠道配置或后台权限接口。

## 范围说明

优先处理由当前代码或默认配置导致的 XSS、敏感信息泄露、文件处理、导入导出、WebDAV 同步、权限控制、额度扣费和供应链风险。

第三方模型服务、托管平台、浏览器插件、用户自行泄露的 API Key、没有实际利用路径的安全建议，通常不属于本项目安全问题范围。
