# 快速开始

## 1. 准备环境

安装 Docker Engine 与 Docker Compose，克隆仓库后进入项目目录：

```bash
git clone https://github.com/Jizhidemu52/Vincent-s-Canvas.git
cd Vincent-s-Canvas
cp .env.example .env
```

编辑 `.env`，至少替换数据库密码、首位管理员初始密码、Provider 加密密钥和对象存储密钥。

## 2. 启动完整服务

```bash
docker compose up -d --build
docker compose ps
curl http://localhost:3000/api/health
```

## 3. 首次管理员登录

1. 打开 `http://localhost:3000/admin/login`。
2. 使用 `.env` 中的超级管理员账号和初始密码。
3. 按提示修改密码。
4. 进入账号管理创建部门和设计师。

## 4. 设计师登录

1. 打开 `http://localhost:3000/login`。
2. 输入管理员创建的中文账号、英文账号、邮箱或工号与密码。
3. 首次登录修改密码。
4. 设计师只能看到自己的项目、任务、素材、历史和剩余额度。

## 5. 配置模型

超级管理员在后台按顺序配置 Provider、工作流、模型和积分价格。API Key 加密保存在服务端，设计师只看到模型名称、能力、启用状态与积分预估。

需要先验证业务链路时，在测试环境设置 `TASK_MOCK_MODE=true`；确认任务、额度、素材和历史后必须在生产环境改回 `false`。

详细步骤和截图见 [完整操作手册](../../../manual/operation-manual.md) 与 [生产部署手册](../../../manual/production-deployment.md)。
