# Docker 部署

根目录 Compose 会启动 Web、API、Worker、PostgreSQL、Redis、MinIO、初始化器和备份服务。

## 启动

```bash
cp .env.example .env
# 编辑生产密码、加密密钥、对象存储和企业微信参数
bun ops/preflight/production-preflight.ts --require-wecom
docker compose up -d --build
docker compose ps
```

只允许 HTTPS 网关访问 Web。不要把 `3100`、`5432`、`6379`、`9000` 或 `9001` 暴露到公网。

## 数据归属

| 服务 | 保存内容 |
| --- | --- |
| PostgreSQL | 用户、部门、权限、项目、账本、任务、历史、素材元数据、审计 |
| Redis | Session、登录限流和任务队列 |
| S3/MinIO | 原图、生成结果、视频、音频、文本和数据库备份 |
| Worker | 从队列取任务，解密服务端 Provider 凭据并执行模型调用 |

## 健康检查与日志

```bash
curl http://localhost:3000/api/health
docker compose logs --tail=200 api worker postgres redis minio backup
```

## 备份恢复

备份容器定期把 PostgreSQL dump 写到对象存储。恢复前停止 API 和 Worker，再使用 `restore.sh` 恢复指定对象。完整命令、企业微信和 40 人验收步骤见 [生产部署与验收手册](../../../manual/production-deployment.md)。
