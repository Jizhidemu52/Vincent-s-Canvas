# 功能介绍

无线画布由设计师创作端、管理员后台和生产业务后端组成。账号、额度、模型、任务、素材、历史和审计使用同一套服务端数据，不依赖前端模拟身份。

## 登录与权限

- 中文账号、英文账号、邮箱或工号加密码登录。
- 新账号首次登录强制修改密码。
- 企业微信扫码登录，并预留 LDAP/OIDC 身份适配接口。
- 超级管理员首次启用 MFA 后才能进入业务后台。
- 超级管理员管理全公司；部门管理员只管理本部门；设计师只看本人数据。
- 停用账号会使已有 Session 失效，所有管理员 API 都在服务端校验角色。

## 账号与额度

- 管理员创建、停用、重置账号，支持 CSV 批量导入约 100 位设计师。
- 设置部门、角色、初始积分、当前余额和额度上限。
- 操作价格与模型价格相加后在前端显示预估。
- 提交任务先冻结积分，成功捕获，失败、取消或超时释放。
- `requestId` 保证幂等，同一任务不会重复扣费。
- 账本记录调整、冻结、结算、释放及操作人。

## Provider、模型与工作流

- 管理 OpenAI 兼容、Gemini、火山引擎、RunningHub、ComfyUI、自定义及内部 Provider。
- API Key 只在管理员端提交，使用服务端加密密钥保存，设计师接口不返回原文。
- 模型配置包含能力、单次积分、人民币成本、并发限制和启用状态。
- 服务端验证任务类型与模型能力，音频任务不能误用图片模型。
- RunningHub、ComfyUI 和自定义工作流统一配置提交路径、状态路径、模板变量、成功/失败值与结果路径。

## 图片、视频与音频

- 文生图、参考图编辑、局部编辑、细节增强、角度控制和无缝拼接。
- 视频工作台支持提示词、参考图、比例、时长、清晰度与模型选择。
- 画布音频节点可提交服务端音频任务并保存结果。
- 图片、视频、音频和工作流任务全部进入 Redis 队列，由 Worker 解密 Provider 凭据后调用上游。
- 设计师浏览器不保存外部模型 API Key，也不直接调用带密钥的上游接口。

## 无限画布与批量改图

- 多项目画布、节点拖拽缩放、连线、撤销重做、导入导出和小地图。
- 图片、文本、视频、音频和生成配置节点可互相连接。
- 批量一键改图支持选择多张图片、统一提示词、逐张排队和逐张状态。
- 单张失败不影响整批；成功项分别进入历史和素材库并单独结算积分。
- 模拟模式仍走真实队列、账本、历史和对象存储，用于不调用模型的业务验收。

## 素材、项目与历史

- 上传图片、视频和文本，生成结果自动进入服务端素材库。
- PostgreSQL 保存素材归属、项目、任务、提示词、模型和来源；S3/MinIO 保存文件内容。
- 设计师 A 无法列表、读取或下载设计师 B 的素材。
- 部门管理员只看本部门，超级管理员可按设计师和部门筛选全公司素材。
- 历史记录保存原图、结果、提示词、模型、参数、积分、人民币成本、状态与失败原因。
- 管理员可导出 CSV 用于财务、审计和内部复盘。

## 审计与运维

- 登录、改密、MFA、账号变更、额度调整、Provider/工作流配置、导出和任务失败均写入审计。
- 系统集成页只显示企业微信、加密密钥和对象存储是否就绪，不返回 Secret。
- Docker Compose 提供 PostgreSQL、Redis、MinIO、Worker、初始化和定时备份。
- GitHub Actions 验证迁移、身份隔离、备份恢复、40 人并发及任务/账本/素材数量一致性。

## 仍需公司环境提供

- 正式域名、HTTPS 网关和 Linux 私有云服务器。
- 企业微信自建应用参数与可信回调域。
- 正式数据库、Redis、对象存储和 Provider 凭据。
- 各真实模型供应商的限流、超时和成本验收。
