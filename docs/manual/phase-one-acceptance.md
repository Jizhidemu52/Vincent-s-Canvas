## 3. 三级权限

| 角色 | 已验证边界 | 结论 |
| --- | --- | --- |
| 设计师 | 后台账号、系统集成和内部 AI 接口均返回 `403`；项目、任务、素材只看本人 | 已通过 |
| 部门管理员 | 账号、项目、任务、历史和素材只限本部门；跨部门修改和重置密码返回 `403` | 已通过 |

权限最终以服务端 Session 和数据库角色为准。隐藏菜单只是界面体验，不能替代 API 鉴权。

## 5. 后续业务能力现状

以下原计划后续阶段也已经进入服务端并通过自动化：

- 幂等额度冻结、成功结算及失败/取消释放。
- Provider 与工作流配置、API Key 加密保存和设计师端密钥清除。
- 图片、视频、音频和批量任务的 Redis Worker 链路。
- 每位设计师的项目、任务、历史和素材隔离。
- S3/MinIO 对象存储、PostgreSQL 备份恢复和审计导出。
- 40 位独立设计师持续提交 2 分钟，失败率低于验收阈值。

## 6. 公司现场待办

这些事项不能用仓库里的测试密钥代替，完成后才能称为公司正式上线：

1. 准备 Linux 私有云服务器、正式域名和 HTTPS 网关。
2. 创建企业微信自建应用，提供 Corp ID、Agent ID、Secret 和可信回调域。
3. 配置生产 PostgreSQL、Redis、S3/MinIO 密码与备份保留策略。
4. 在管理员后台录入真实 Provider Key，并逐个验证限流、超时、失败退款和人民币成本。
5. 使用 10 位真实设计师试点，核对企业微信登录、部门权限、素材隔离和审计导出。
6. 完成一次生产备份恢复演练后，再扩展到约 100 位设计师。
